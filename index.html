Monitoring Overview

As part of our cloud migration, we've implemented monitoring to ensure system performance, security, and availability.

  • Tools used: AWS CloudWatch, Azure Monitor, Datadog
  • Monitored Resources: EC2, RDS, S3, network traffic
  • Frequency: Metrics collected every 1–5 minutes
  • Testing: Alerts can be tested using load generators or fault injection
  • Adjustments: Thresholds reviewed monthly; config stored in Git

Alerting & Notifications

Alerts notify the right teams when an issue is detected. Alerts are tiered by severity:

  • Critical: Immediate pager alert via PagerDuty
  • Warning: Slack + email to the ops channel
  • Info: Logged in dashboard only

Notification channels include: Slack, PagerDuty, and email groups.

Reporting

Cloud activity and performance reports help us track trends and ensure compliance.

  • Daily dashboard in Datadog with uptime and latency trends
  • Weekly email summary via AWS SNS + Lambda
  • Custom reports can be scheduled via PowerBI or emailed on demand

FAQ

  • How do I subscribe to alerts? Contact ops@company.com or request access via internal portal
  • Can I pause alerts? Yes, alerts can be silenced via the alert dashboard for maintenance windows
  • How do I add a new service to monitoring? Open a Jira ticket or follow the monitoring-onboarding guide
